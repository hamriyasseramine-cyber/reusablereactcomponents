# Why use React

Building interfaces with plain JavaScript quickly becomes messy: you end up manually finding elements, updating their content, and keeping track of what changed. React solves this by letting you describe *what* the UI should look like for a given state, and it figures out *how* to update the actual page for you.

On top of that, React's component model makes it easy to reuse pieces of UI across your app, which keeps large projects organized and consistent instead of turning into a pile of duplicated code.
